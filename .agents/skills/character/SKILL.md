# Character — робота з персонажами

## Контекст

Цей скіл працює з картками персонажів у директорії `characters/`. Кожен персонаж — окремий md-файл. Усі згадки персонажа в сценах мають бути через `[[wikilinks]]`.

### Релевантні директорії та шаблони

| Директорія | Призначення | Шаблон |
|---|---|---|
| `characters/` | Картки персонажів | `character.md` |
| `scenes/` | Сцени з персонажем | `scene.md` |
| `events/` | Події за участю персонажа | `event.md` |

### Шаблон character.md — frontmatter

```yaml
---
title: "{{title}}"
date: {{format-date now}}
role: ""
arc: ""
motivation: ""
conflict: ""
aliases: []
tags: [character]
---
```

- `role` — тільки одне з: `protagonist`, `antagonist`, `supporting`
- `arc` — тип арки (redemption, fall, growth, flat, corruption)
- `motivation` — що рухає персонажем (внутрішнє бажання)
- `conflict` — внутрішній конфлікт (чого персонаж хоче vs чого потребує)
- `aliases` — альтернативні імена, прізвиська для пошуку

### Ключові команди

```bash
just characters          # список усіх персонажів (ім'я, роль, арка)
zk backlinks <slug>      # усі сцени/події, що посилаються на персонажа
just search "ім'я"       # пошук згадок персонажа по всіх файлах
```

## Процес

### Крок 1: Створення нового персонажа

1. Отримай від користувача:
   - Ім'я персонажа
   - Роль (protagonist / antagonist / supporting)
   - Контекст: хто він у світі, яка функція в сюжеті

2. **Перевір унікальність:** знайди схожі імена через `just search "ім'я"`. Якщо ім'я вже зайняте — попередь користувача.

3. Згенеруй slug з імені (транслітерація, малі літери, дефіси).

4. Створи `characters/<slug>.md` з шаблоном `character.md`.

### Крок 2: Заповнення картки персонажа

У **frontmatter** обов'язково заповни:
- `title` — повне ім'я персонажа
- `role` — protagonist / antagonist / supporting
- `motivation` — 1 речення про ключове бажання
- `conflict` — 1 речення про внутрішній конфлікт
- `arc` — тип арки (якщо визначено)
- `aliases` — скорочені імена, прізвиська

У **тілі файлу** опиши:

```markdown
## Зовнішність
2-3 ключові деталі зовнішності.

## Характер
Основні риси, манера поведінки, звички.

## Історія
Передісторія до початку книги. Звідки родом, ключові події минулого.

## Стосунки
- [[інший-персонаж]] — характер стосунків
- [[ще-один]] — ...

## Роль у сюжеті
Яку функцію виконує в історії.
```

### Крок 3: Зв'язування з існуючим контентом

1. Додай wikilinks у тілі картки на: інших персонажів (у розділі «Стосунки»), локації (`[[world/...]]`), події (`[[events/...]]`).
2. Якщо персонаж уже згадується в сценах — запропонуй оновити згадки: додати `[[wikilinks]]` навколо імені у відповідних сценах.

### Крок 4: Оновлення існуючого персонажа

Якщо користувач просить оновити персонажа:
1. Прочитай `characters/<slug>.md`
2. Знайди всі сцени з персонажем через `zk backlinks`
3. Внеси зміни в картку
4. Перевір, чи зміни не конфліктують з написаними сценами
5. Якщо конфліктують — попередь і запропонуй варіанти

## Правила

1. **Кожен персонаж — окремий файл.** Не створюй збірних карток на кількох персонажів.
2. **Перевіряй унікальність імені** перед створенням. Два персонажі з однаковим іменем = dead wikilinks і плутанина.
3. **`role` — суворо зі списку:** `protagonist`, `antagonist`, `supporting`. Якщо роль змінюється впродовж історії — вкажи основну на початок книги.
4. **`aliases` — для пошуку.** Сюди потрапляють скорочення, прізвиська, альтернативні написання. Це допомагає `zk backlinks` і `just search` знаходити згадки.
5. **Не вигадуй події без потреби.** Якщо в передісторії персонажа з'являється нова подія — запропонуй створити файл у `events/`.
6. **Показуй звіт** після кожної дії: що створено, які поля заповнені, які wikilinks додано, скільки сцен посилаються.

## Приклади

### Приклад 1: Новий персонаж

> **Користувач:** Створи персонажа — старого мудреця на ім'я Мирон, який живе біля [[dark-forest]] і знає таємницю [[the-ritual]]. Він допомагає головному герою.

**Дії агента:**
1. Перевіряє ім'я «Мирон» через `just search "мирон"` — не знайдено
2. Генерує slug: `myron`
3. Створює `characters/myron.md`:
   ```yaml
   ---
   title: "Мирон"
   date: 2026-07-12
   role: supporting
   arc: mentor
   motivation: "Захистити світ від повторення [[the-ritual]]"
   conflict: "Знає правду, але не може розкрити її одразу — повинен підвести [[vlad]] до самостійного відкриття"
   aliases: [старий, мудрець, відлюдник]
   tags: [character]
   ---
   ```
4. Заповнює тіло (зовнішність, характер, історія, стосунки з [[vlad]], роль)
5. Звітує: «Створено `characters/myron.md`, роль: supporting, арка: mentor»

### Приклад 2: Розвиток персонажа

> **Користувач:** Я хочу змінити роль [[marta]] з supporting на protagonist. І дати їй арку redemption.

**Дії агента:**
1. Читає `characters/marta.md`
2. Знаходить усі сцени з [[marta]] через `zk backlinks marta`
3. Змінює `role` на `protagonist`, `arc` на `redemption`
4. Оновлює опис у тілі: додає redemption-елементи в історію та стосунки
5. Попереджає: «Змінено роль на protagonist. Перевір 5 сцен — чи не потрібно додати POV Марти там, де вона тепер головна?»

### Приклад 3: Стосунки

> **Користувач:** Опиши стосунки між [[vlad]] і [[ostap]]. Вони брати, але ворогують через спадок.

**Дії агента:**
1. Читає обидва файли: `characters/vlad.md`, `characters/ostap.md`
2. Оновлює розділ «Стосунки» в обох:
   - У `vlad.md`: `- [[ostap]] — молодший брат, ворожнеча через спадок [[old-mill]]`
   - У `ostap.md`: `- [[vlad]] — старший брат, ворожнеча через спадок [[old-mill]]`
3. Перевіряє консистентність: чи не описані вони як друзі в якійсь сцені
4. Звітує: «Оновлено стосунки в обох картках. Додано wikilinks на [[old-mill]].»
