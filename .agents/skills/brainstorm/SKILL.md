# Brainstorm — генерація ідей

## Контекст

Цей скіл працює в межах zk-нотатника для художньої книги.

### Релевантні директорії

| Директорія | Призначення | Шаблон |
|---|---|---|
| `inbox/` | Сирі ідеї, швидкі нотатки | `default.md` |
| `notes/` | Оформлені ідеї, концепції | `note.md` (type: idea) |
| `scenes/` | Сцени (НЕ писати без явного запиту) | `scene.md` |
| `chapters/` | Глави-збірки | `chapter.md` |
| `characters/` | Картки персонажів | `character.md` |
| `world/` | Локації | `location.md` |
| `events/` | Події | `event.md` |

### Ключові команди

```bash
just search "запит"    # пошук по всіх md-файлах
zk backlinks <slug>    # хто посилається на нотатку
zk list --tags idea    # усі нотатки з тегом #idea
```

## Процес

### Крок 1: Зрозуміти контекст

1. Уточни в користувача, яка саме частина історії потребує ідей:
   - Певна сцена чи глава?
   - Персонаж і його арка?
   - Сюжетний поворот?
   - Загальний напрямок?
2. Прочитай релевантні файли:
   - Згадані сцени, персонажі, події
   - Сусідні сцени (до/після) для контексту
   - Загальний індекс (`index.md`)

### Крок 2: Згенерувати варіанти

Запропонуй **3–5 варіантів**. Кожен варіант має містити:

- **Назва** — коротке формулювання ідеї
- **Суть** — 2–4 речення про те, що станеться
- **Плюси** — чому це добре для історії (посилює арку, додає напруги, розкриває персонажа)
- **Мінуси** — ризики, потенційні проблеми, складнощі реалізації
- **Наслідки** — як це вплине на подальший сюжет, інших персонажів

Форматуй варіанти з wikilinks на існуючі файли:

```markdown
### Варіант 1: «Влад знаходить лист»

**Суть:** Під час обшуку [[old-mill]] [[vlad]] знаходить лист від [[marta]],
який розкриває справжню причину [[the-fire]].

**Плюси:** Розкриває мотивацію [[marta]], створює конфлікт між [[vlad]] і [[ostap]].

**Мінуси:** Може передчасно розкрити таємницю, якщо не обмежити зміст листа.

**Наслідки:** [[vlad]] має вирішити — показати лист [[ostap]] чи приховати.
Це запускає арку недовіри, яка тягнеться до [[ch03-sc05]].
```

### Крок 3: Зберегти (опційно)

Якщо користувач хоче зберегти ідею:

- **Сира ідея** → створи `inbox/<slug>.md` з шаблоном `default.md`, тег `#idea`
- **Оформлена ідея** → створи `notes/<slug>.md` з шаблоном `note.md`, `type: idea`, тег `#idea`

Додай wikilinks на пов'язані файли. Покажи звіт про створений файл.

## Правила

1. **Не писати в `scenes/` без явного запиту користувача.** Brainstorm — це ідеї, не готова проза.
2. **Не суперечити існуючому лору.** Перед генерацією ідей прочитай релевантні файли, щоб не запропонувати те, що вже неможливо.
3. **Пропонуй різноманітні варіанти**, а не варіації одного. Різні жанрові ходи, різні наслідки, різні персонажі у фокусі.
4. **Використовуй wikilinks** (`[[slug]]`) для всіх згадок персонажів, локацій, подій, сцен.
5. **Позначай створені файли** тегом `#idea` та wikilinks на джерело натхнення.

## Приклади

### Приклад 1: Сюжетний глухий кут

> **Користувач:** Як мені вийти з ситуації, де [[vlad]] загнаний у кут у сцені [[caught-in-alley]]? Він не може битися, не може втекти.

**Дії агента:**
1. Читає `scenes/caught-in-alley.md` — розуміє контекст
2. Читає `characters/vlad.md` — знає його навички, слабкості
3. Читає `world/` якщо є локація провулку — знає оточення
4. Генерує 3–5 варіантів (наприклад: несподівана допомога, використання оточення, блеф)
5. Кожен з wikilinks, плюсами/мінусами, наслідками

### Приклад 2: Арка персонажа

> **Користувач:** Мені потрібна ідея для арки [[marta]]. Вона другорядна, але я хочу дати їй більше глибини.

**Дії агента:**
1. Читає `characters/marta.md` — поточна роль, мотивація
2. Шукає всі згадки через `just search "marta"` — де вона з'являється
3. Пропонує 3–5 варіантів арок: redemption, corruption, mentor, secret keeper
4. Для кожного — як це вплине на сюжет, які сцени треба додати/змінити

### Приклад 3: Порожній старт

> **Користувач:** Я хочу почати нову історію. Дай мені 3 концепції для фентезі-детективу в світі, де магія щойно зникла.

**Дії агента:**
1. Генерує 3 концепції (не читаючи файлів, бо їх ще немає)
2. Для кожної: логлайн, головний герой, центральний конфлікт, тон
3. Пропонує створити `index.md` з описом концепції та `characters/` для головного героя
