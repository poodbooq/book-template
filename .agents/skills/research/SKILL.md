# Research — дослідження

## Контекст

Цей скіл виконує дослідження реальних тем для художньої книги. Результати зберігаються в `notes/` як структуровані нотатки. Не плутати з `events/` — research це про **реальний світ**, events — про **світ книги**.

### Релевантні директорії та шаблони

| Директорія | Призначення | Шаблон |
|---|---|---|
| `notes/` | Результати досліджень | `note.md` |
| `scenes/` | Сцени, для яких робиться дослідження | `scene.md` |

### Шаблон note.md — frontmatter

```yaml
---
title: "{{title}}"
created: {{format-date now}}
updated: {{format-date now}}
type: research          # research або reference
sources: []
aliases: []
tags: []
---
```

- `type: research` — власне дослідження, аналіз, висновки
- `type: reference` — стороннє джерело, цитата, посилання

### Ключові команди

```bash
just search "запит"          # чи є вже нотатки на цю тему?
```

Для інтернет-пошуку використовуй веб-пошук (web search) для пошуку релевантних джерел та веб-завантаження (web fetch) для отримання повного тексту сторінок.

## Процес

### Крок 1: Зрозуміти тему

1. Уточни в користувача:
   - **Що досліджуємо?** Конкретна тема (металургія XIII ст., римське право, симптоми отруєння)
   - **Для чого?** Яка сцена/персонаж/подія цього потребує
   - **Глибина?** Поверхневий огляд чи детальне дослідження?

2. Перевір, чи є вже нотатки на цю тему:
   ```bash
   just search "ключове слово"
   ```

### Крок 2: Пошук інформації

1. **В інтернеті** — виконай веб-пошук для пошуку релевантних джерел:
   - Формулюй запит англійською для кращих результатів
   - Шукай авторитетні джерела: Wikipedia, наукові статті, історичні довідники
   - Запитуй 5–10 результатів

2. **Заглиблення** — для найкращих результатів завантаж повний текст сторінок.

### Крок 3: Структурувати результати

Створи `notes/<slug>.md` з шаблоном `note.md`. Заповни frontmatter:
- `title` — назва теми дослідження
- `type: research` — якщо це аналіз/висновки
- `type: reference` — якщо це збірка джерел/цитат
- `sources` — список URL або бібліографічних посилань
- `tags` — тематичні теги (напр. `#history`, `#medicine`, `#weapons`)

У тілі файлу:
```markdown
## Ключові факти
Список основних знахідок, важливих для книги.

## Застосування до книги
Як ці факти впливають на:
- Сцену [[forging-sword]]
- Персонажа [[vlad]]
- Світ загалом

## Джерела
- [Назва джерела](URL) — короткий опис
- ...
```

### Крок 4: Зв'язати з контентом книги

1. Додай wikilinks на релевантні сцени, персонажів, події
2. Запропонуй, які сцени можуть потребувати оновлення на основі нових знань
3. Якщо дослідження суперечить уже написаному — попередь

## Правила

1. **Розрізняй `type: research` і `type: reference`.**
   - `research` — ти проаналізував інформацію і зробив висновки
   - `reference` — ти зібрав джерела, цитати, посилання (без глибокого аналізу)

2. **Завжди вказуй `sources`.** Користувач має знати, звідки інформація. Для веб-сторінок — повний URL; не вигадуй точні медичні чи історичні твердження без джерела.

3. **Не плутай research з events.** Research — реальний світ (як працює середньовічна металургія). Events — світ книги (подія «Кування Меча»).

4. **Факти > думки.** Опирайся на авторитетні джерела. Якщо інформація суперечлива — зазнач це.

5. **Застосування до книги — обов'язково.** Недостатньо просто зібрати факти. Поясни, як вони впливають на конкретні сцени, персонажів, світ.

6. **Не перевантажуй.** Якщо тема велика — розбий на кілька нотаток. Одна нотатка = одна тема.

## Приклади

### Приклад 1: Історичне дослідження

> **Користувач:** Досліди середньовічну металургію для сцени [[forging-sword]].

**Дії агента:**
1. Перевіряє наявність нотаток: `just search "металургія"` — немає
2. Шукає в інтернеті: "medieval metallurgy swordsmithing process 13th century"
3. Отримує 5 релевантних джерел, завантажує 2 найкращих
4. Створює `notes/medieval-metallurgy.md`:
   ```yaml
   ---
   title: "Середньовічна металургія"
   created: 2026-07-12
   updated: 2026-07-12
   type: research
   sources: ["https://en.wikipedia.org/wiki/Medieval_metallurgy", "https://..."]
   aliases: [ковальство, кування меча]
   tags: [history, weapons, research]
   ---
   ```
5. Заповнює тіло: ключові факти, процес кування, матеріали, час
6. Застосування: «Сцена [[forging-sword]]: виправити — меч не можна скувати за одну ніч. Процес займає 2–3 тижні. Пропоную розтягнути в часі або показати лише фінальний етап.»
7. Додає wikilinks: `[[forging-sword]]`, `[[vlad]]`, `[[old-mill]]` (якщо там кузня)
8. Звітує: «Створено `notes/medieval-metallurgy.md`. Знайдено 5 джерел. Сцена [[forging-sword]] потребує коригування хронології.»

### Приклад 2: Медичне дослідження

> **Користувач:** Як виглядає отруєння беладоною? Мені потрібно для сцени, де [[marta]] отруюють.

**Дії агента:**
1. Шукає: "belladonna poisoning symptoms timeline deadly nightshade"
2. Завантажує медичні джерела
3. Створює `notes/belladonna-poisoning.md`, `type: research`
4. Описує симптоми по годинах: 0–2 год — сухість у роті, 2–6 год — галюцинації, 6–24 год — кома
5. Застосування: «У сцені отруєння [[marta]] симптоми мають проявлятися поступово. Неправильно: впала одразу. Правильно: спершу розширені зіниці, потім сплутана мова.»
6. Звітує з конкретними рекомендаціями для сцени

### Приклад 3: Збірка референсів

> **Користувач:** Збери референси для опису [[dark-forest]] — мені потрібні фото і описи моторошних лісів.

**Дії агента:**
1. Шукає: "haunted forest descriptions gothic literature atmosphere" → 10 результатів
2. Для кожного цікавого — завантажує повний текст
3. Створює `notes/dark-forest-references.md`, `type: reference`
4. Групує: атмосферні описи, візуальні деталі, звуки, запахи
5. Кожен референс — з URL джерела і короткою цитатою
6. Звітує: «Зібрано 7 референсів у `notes/dark-forest-references.md`»
