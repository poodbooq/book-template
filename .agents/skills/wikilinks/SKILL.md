# Wikilinks — робота з посиланнями

## Контекст

Цей скіл додає, виправляє та аналізує `[[wikilinks]]` у md-файлах zk-нотатника. Wikilinks зв'язують сцени, персонажів, локації, події в єдину мережу, що дозволяє навігувати рукописом через zk і zk-nvim.

### Як виглядають wikilinks

- Звичайне посилання: `[[slug]]` або `[[slug|текст]]`
- Посилання в главі: `- [[slug]]` — задає порядок сцен і зберігає backlinks у `zk`
- Slug — це назва файлу без розширення `.md`

Приклад: файл `characters/vlad.md` → wikilink: `[[vlad]]`

### Релевантні директорії

| Директорія | Приклад slug | Що шукати в тексті |
|---|---|---|
| `characters/` | `vlad`, `marta`, `ostap` | Імена персонажів |
| `world/` | `old-mill`, `dark-forest` | Назви локацій |
| `events/` | `the-fire`, `the-ritual` | Назви подій |
| `scenes/` | `ch01-sc01`, `forging-sword` | Назви сцен |
| `notes/` | `magic-system` | Ключові концепції |

### Ключові команди

```bash
zk backlinks <slug>                    # усі файли, що посилаються на slug
zk orphans                             # нотатки без жодного вхідного посилання
just characters                        # список усіх персонажів
just search "текст"                    # пошук згадок по всіх файлах
```

### Як визначити slug з назви файлу

Slug — це назва файлу без `.md`. Наприклад:
- Файл `characters/vlad.md` → slug `vlad`
- Файл `world/old-mill.md` → slug `old-mill`
- Файл `events/the-fire.md` → slug `the-fire`

Щоб знайти всі доступні slugs, які можна використовувати:
```bash
find . -name '*.md' -not -path './.git/*' | sed 's|.*/||;s|\.md$||' | sort
```

## Процес

### Крок 1: Визначити цільовий файл

Отримай від користувача, який файл (або файли) потрібно обробити:

- Конкретний файл: «Додай wikilinks у [[ch01-sc01]]»
- Усі файли в директорії: «Розстав wikilinks у всіх сценах»
- Увесь рукопис: «Зв'яжи все через wikilinks»

Якщо файл не вказано — запитай.

### Крок 2: Зібрати інвентар доступних slugs

Перед аналізом склади список усіх доступних wikilinks:

```bash
find . -name '*.md' -not -path './.git/*' | sed 's|.*/||;s|\.md$||' | sort
```

Збережи в пам'яті. Це твій інвентар — ти можеш посилатися тільки на ті slugs, які реально існують.

### Крок 3: Проаналізувати текст цільового файлу

1. Прочитай файл
2. Розділи текст на:
   - **Frontmatter** (між `---` i `---`) — НЕ додаємо wikilinks у frontmatter
   - **Заголовки** (рядки що починаються з `#`) — НЕ вікіфікуємо
   - **Тіло** — аналізуємо на наявність потенційних wikilinks

3. Для кожного абзацу тіла знайди:
   - Імена персонажів → порівняй з `characters/*.md`
   - Назви локацій → порівняй з `world/*.md`
   - Назви подій → порівняй з `events/*.md`
   - Назви інших сцен → порівняй з `scenes/*.md`
   - Ключові концепції → порівняй з `notes/*.md`

### Крок 4: Порівняння з існуючими wikilinks

Для кожної знахідки перевір:
1. Чи є вже `[[...]]` навколо цього тексту? → якщо так, пропусти
2. Чи є файл з таким slug в інвентарі? → якщо так, можна додавати
3. Чи це перше входження терміну в тексті? → тільки перше входження

### Крок 5: Додати або виправити wikilinks

**Додавання:**
- Знайшов текст, що відповідає slug → обгорни в `[[slug]]`
- Зроби точкову заміну тексту на `[[slug]]`
- Приклад: `Влад стояв` → `[[vlad]] стояв`

**Виправлення (мертві wikilinks):**
- Знайшов `[[old-slug]]`, але файл `old-slug.md` не існує
- Знайди правильний slug (можливо, файл перейменували)
- Якщо є схожий: `[[old-slug]]` → `[[new-slug]]`
- Якщо немає: познач у звіті як `[warn]`

**Виправлення (alias):**
- Якщо персонаж має `aliases` у frontmatter — перевір, чи використовуються аліаси в тексті без wikilinks
- Приклад: «старий» → `[[myron|старий]]` (якщо `myron` має alias «старий»)

### Крок 6: Звіт

Покажи підсумок:

```markdown
## Wikilinks: звіт

**Файл:** `scenes/ch02-sc03.md`

### Додано (3)
- `[[vlad]]` — рядок 5
- `[[old-mill]]` — рядок 12
- `[[the-fire]]` — рядок 18

### Виправлено (1)
- `[[old-sage]]` → `[[myron]]` — рядок 8

### Пропущено (2)
- «Марта» — рядок 7, вже є `[[marta]]` у рядку 3 (не перше входження)
- «темний ліс» — рядок 20, файл `dark-forest.md` існує, але wikilink не додано
  бо цей slug відрізняється від тексту (текст українською, slug англійською).
  Пропозиція: використати `[[dark-forest|темний ліс]]`
```

## Правила

1. **Тільки перше входження.** Wikilink додається лише навколо **першого** входження терміну в тексті (не враховуючи заголовки). Наступні згадки — звичайний текст.

2. **Не вікіфікувати заголовки.** Рядки, що починаються з `#`, `##`, `###` тощо — заголовки, wikilinks не додаються.

3. **Не вікіфікувати всередині існуючих wikilinks.** Текст `[[vlad]]` не потребує додаткових wikilinks всередині.

4. **Не вікіфікувати у frontmatter.** Все між `---` i `---` на початку файлу — frontmatter, не чіпати.

5. **Перевіряти існування файлу.** Перед додаванням wikilinks переконайся, що файл існує. Список slugs з Кроку 2 — твоє джерело правди.

6. **Slug vs текст.** Якщо slug файлу англійською, а текст українською — використовуй pipe-синтаксис: `[[dark-forest|темний ліс]]`.

7. **Не додавай wikilinks на сам файл.** Якщо обробляєш `scenes/vlad-meets-marta.md`, не додавай `[[vlad-meets-marta]]` всередину нього ж.

8. **Показуй звіт** після кожної дії: що додано, що виправлено, що пропущено і чому.

9. **Не змінюй текст.** Wikilinks додаються навколо існуючого тексту, не змінюючи його. Виняток — виправлення битих посилань.

10. **Не використовуй `![[...]]`.** `zk` 0.15.x не індексує Obsidian embeds як backlinks. У главах використовуй `- [[scene-slug]]`; exporter сам вставить текст сцени.

## Приклади

### Приклад 1: Додавання wikilinks у сцену

> **Користувач:** Додай wikilinks у сцену [[ch01-sc01]].

**Дії агента:**
1. Збирає інвентар slugs
2. Читає `scenes/ch01-sc01.md`
3. Аналізує текст:
   - «Влад стояв біля старого млина» → `vlad` є в інвентарі, wikilinks немає → додати `[[vlad]]`
   - «старий млин» → `old-mill` є в інвентарі → додати `[[old-mill|старий млин]]`
   - «Він згадав пожежу» → `the-fire` є в інвентарі → додати `[[the-fire|пожежу]]`
   - «Влад попрямував далі» → друге входження, не додаємо
4. Виконує заміну для кожного
5. Звітує: «Додано 3 wikilinks: [[vlad]], [[old-mill|старий млин]], [[the-fire|пожежа]]»

### Приклад 2: Виправлення мертвих wikilinks

> **Користувач:** Виправ wikilinks у [[ch02-sc05]]. Там щось не працює.

**Дії агента:**
1. Збирає інвентар slugs
2. Читає `scenes/ch02-sc05.md`
3. Знаходить `[[the-old-man]]` — перевіряє інвентар: файл `the-old-man.md` не існує
4. Шукає схожі: є `myron.md`, який в аліасах має «старий»
5. Виправляє: `[[the-old-man]]` → `[[myron|старий]]`
6. Знаходить `[[dark-castle]]` — файл не існує, схожих немає
7. Звітує:
   - Виправлено: `[[the-old-man]]` → `[[myron|старий]]`
   - `[warn]` `[[dark-castle]]` — файл не знайдено. Можливо, ще не створено?

### Приклад 3: Повний аналіз зв'язків

> **Користувач:** Проаналізуй зв'язки для [[vlad]]. З ким він пов'язаний, де з'являється, які події?

**Дії агента:**
1. Запускає `zk backlinks vlad` → отримує всі файли, що посилаються на [[vlad]]
2. Групує за типом:
   - **Сцени (8):** `[[ch01-sc01]]`, `[[ch02-sc03]]`, ...
   - **Події (2):** `[[the-fire]]` (свідок), `[[the-ritual]]` (учасник)
   - **Персонажі (3):** `[[marta]]`, `[[ostap]]`, `[[myron]]`
   - **Локації (2):** `[[old-mill]]`, `[[village]]`
3. Перевіряє, чи [[vlad]] посилається на них у відповідь (через `zk backlinks` для кожного)
4. Звітує: «[[vlad]] має 15 зв'язків. Усі взаємні. Пропущених немає.»

### Приклад 4: Зв'язування персонажа зі сценами

> **Користувач:** Персонаж [[myron]] згадується в сценах, але без wikilinks. Виправ.

**Дії агента:**
1. Читає `characters/myron.md` — знає slug, aliases: [старий, мудрець, відлюдник]
2. Запускає `just search "мирон"` і `just search "старий"` і `just search "мудрець"` — знаходить усі згадки
3. Для кожної сцени, де є згадка без wikilinks:
   - Перевіряє, чи є вже `[[myron]]` — якщо ні, додає на перше входження
   - Якщо текст містить аліас («старий») — використовує `[[myron|старий]]`
4. Звітує: «Додано wikilinks у 5 сцен. 2 сцени вже мали wikilinks — пропущено.»
