# Write Scene — написання сцен

## Контекст

Цей скіл працює зі сценами художньої книги. Сцени — це атомарні md-файли в директорії `scenes/`. Глави (`chapters/`) є впорядкованими списками звичайних `[[wikilinks]]` на сцени і **не містять тексту сцен**.

### Релевантні директорії та шаблони

| Директорія | Призначення | Шаблон | Що робити |
|---|---|---|---|
| `scenes/` | Текст сцен | `scene.md` | Писати, редагувати |
| `chapters/` | Глави-збірки | `chapter.md` | Лише додавати `- [[...]]`, не писати текст |
| `characters/` | Персонажі | `character.md` | Читати для консистентності |
| `world/` | Локації | `location.md` | Читати для контексту |
| `events/` | Події | `event.md` | Читати для хронології |

### Шаблон scene.md — frontmatter

```yaml
---
title: "{{title}}"
created: {{format-date now}}
updated: {{format-date now}}
status: draft
word-count: 0
pov: ""
setting: ""
story-date: ""
timeline-order: 0
act: ""
tags: []
---
```

### Ключові команди

```bash
just wordcount          # оновити word-count у всіх сценах
just list-drafts        # показати сцени/глави з status: draft
just search "запит"     # пошук по тексту
just act 1              # сцени з акту 1
just pov vlad           # сцени від POV персонажа
just setting old-mill   # сцени в локації
```

Стилістичний довідник — див. [references/style-guide.md](references/style-guide.md).

## Процес

### Крок 1: Нова сцена чи редагування?

**Нова сцена:**
1. Отримай від користувача: назву, контекст (що має статися, де, з ким)
2. Згенеруй slug з назви (транслітерація, малі літери, дефіси)
3. Створи файл `scenes/<slug>.md` з шаблоном `scene.md`

**Редагування існуючої:**
1. Прочитай файл сцени
2. Зрозумій, що саме треба змінити
3. Зроби точкові зміни або перезапиши файл повністю

### Крок 2: Зібрати контекст

Перед написанням прочитай пов'язані файли:

1. **Персонажі:** для кожного персонажа в сцені → `characters/<slug>.md`
   - Зовнішність, характер, мотивація, манера говорити
   - Поточний стан у сюжеті (що сталося до цієї сцени)
2. **Локація:** якщо вказана → `world/<slug>.md`
   - Опис, атмосфера, ключові деталі
3. **Події:** якщо сцена є наслідком події → `events/<slug>.md`
4. **Сусідні сцени:** сцени до і після в цій главі → прочитай главу з `chapters/`

### Крок 3: Написати текст

**Формат прози:**
- Звичайний markdown
- Абзаци розділяються порожнім рядком
- Діалоги — кожна репліка з нового рядка, через тире (`—`)
- Внутрішній монолог — курсивом (`*думка*`) або окремим абзацом

**Обов'язково:**
- Використовуй `[[wikilinks]]` для першого входження кожного персонажа, локації, події
- Дотримуйся стилю з [references/style-guide.md](references/style-guide.md)
- Враховуй POV персонажа (якщо вказано в frontmatter)
- Підтримуй тон і темп, узгоджений з сусідніми сценами

### Крок 4: Оновити метадані

Після написання:
1. Порахуй слова в тілі сцени (без frontmatter) і запиши в `word-count: N`
2. Заповни поля, якщо вони порожні: `pov`, `setting`, `story-date`, `timeline-order`, `act`
3. Якщо сцена готова до читання → `status: revised`
4. Якщо це фінальна версія → `status: final` (лише за явним запитом)

### Крок 5: Прив'язати до глави

Запропонуй користувачу виконати `just add-scene <chapter-slug> <scene-slug>`. Не додавай автоматично — спитай, у яку главу.

## Правила

1. **Завжди писати в `scenes/`.** Ніколи не пиши текст сцени прямо в `chapters/`. Глави — це лише списки `- [[scene-slug]]`.
2. **Завжди читати контекст перед написанням.** Персонажі, локації, події — щоб уникнути протиріч.
3. **Не змінювати статус на `final` без явного запиту.** `status: draft` → `status: revised` можна пропонувати. `status: final` — тільки коли користувач скаже «фіналізуй».
4. **Використовувати `[[wikilinks]]`** для першого входження кожного персонажа, локації чи події. Повторні згадки в тій самій сцені лишаються звичайним текстом.
5. **Дотримуватися tone of voice.** Якщо не впевнений у стилі — запитай користувача або перечитай сусідні сцени для калібрування.
6. **Діалоги з нового рядка.** Кожна репліка — окремий рядок. Не змішуй дію і діалог в одному абзаці без потреби.
7. **Показуй звіт після кожної дії:** що створено, що змінено, скільки слів, які wikilinks додано.

## Приклади

### Приклад 1: Нова сцена

> **Користувач:** Напиши сцену, де [[vlad]] зустрічає [[marta]] біля [[old-mill]] після [[the-fire]]. Акт 2, POV Влада.

**Дії агента:**
1. Читає `characters/vlad.md`, `characters/marta.md`, `world/old-mill.md`, `events/the-fire.md`
2. Визначає slug: `vlad-meets-marta-at-mill`
3. Створює `scenes/vlad-meets-marta-at-mill.md` з frontmatter:
   ```yaml
   ---
   title: "Зустріч біля млина"
   created: 2026-07-12
   updated: 2026-07-12
   status: draft
   word-count: 0
   pov: "[[vlad]]"
   setting: "[[old-mill]]"
   story-date: "після [[the-fire]]"
   timeline-order: 20
   act: "2"
   tags: [act2]
   ---
   ```
4. Пише текст сцени (~300–500 слів) з діалогами, описами, wikilinks
5. Рахує слова → оновлює `word-count`
6. Звітує: «Створено `scenes/vlad-meets-marta-at-mill.md`, 420 слів, status: draft. Додати в главу?»

### Приклад 2: Дописати сцену

> **Користувач:** Допиши сцену [[ch02-sc03]], де [[ostap]] заходить у [[dark-forest]].

**Дії агента:**
1. Читає `scenes/ch02-sc03.md`
2. Читає `characters/ostap.md`, `world/dark-forest.md`
3. Дописує продовження після останнього абзацу
4. Оновлює `word-count`
5. Звітує: «Додано 180 слів. Загалом 650 слів.»

### Приклад 3: Переписати фрагмент

> **Користувач:** Перепиши діалог між [[vlad]] і [[marta]] у [[ch01-sc02]]. Зараз він надто мелодраматичний, зроби стриманішим.

**Дії агента:**
1. Читає `scenes/ch01-sc02.md`
2. Знаходить діалогові репліки між персонажами
3. Переписує діалог: коротші речення, менше емоційних вигуків, більше підтексту
4. Виконує заміну
5. Звітує з before/after порівнянням
