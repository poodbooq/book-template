# Worldbuilding — світобудова

## Контекст

Цей скіл працює з елементами вигаданого світу. Локації зберігаються в `world/`, абстрактні концепції (магія, культура, релігія, історія) — у `notes/`.

### Релевантні директорії та шаблони

| Директорія | Призначення | Шаблон |
|---|---|---|
| `world/` | Конкретні локації | `location.md` |
| `notes/` | Концепції, системи, культури | `note.md` (type: worldbuilding) |
| `events/` | Історичні події світу | `event.md` |
| `scenes/` | Сцени в цих локаціях | `scene.md` |

### Шаблон location.md — frontmatter

```yaml
---
title: "{{title}}"
created: {{format-date now}}
updated: {{format-date now}}
type: ""
era: ""
first-appearance: ""
tags: [location]
---
```

- `type` — city / region / building / natural / realm / dungeon
- `era` — епоха, коли існувала (якщо світ має хронологію)
- `first-appearance` — wikilink на сцену, де локація вперше з'являється

### Шаблон note.md — frontmatter (для концепцій)

```yaml
---
title: "{{title}}"
created: {{format-date now}}
updated: {{format-date now}}
type: worldbuilding
sources: []
aliases: []
tags: []
---
```

### Ключові команди

```bash
just setting <loc>      # сцени в локації
just search "назва"     # пошук згадок по всіх файлах
zk backlinks <slug>     # хто посилається на локацію/концепцію
```

## Процес

### Крок 1: Визначити тип контенту

Запитай у користувача або визнач сам:

- **Локація** (місто, будівля, регіон, природний об'єкт) → `world/<slug>.md`, шаблон `location.md`
- **Концепція** (магічна система, релігія, культура, мова, технологія) → `notes/<slug>.md`, шаблон `note.md`, `type: worldbuilding`
- **Історична подія** (битва, падіння імперії, народження героя) → `events/<slug>.md`, шаблон `event.md`

Якщо сумніваєшся — запитай користувача.

### Крок 2: Зібрати контекст

Перед створенням перевір:

1. Чи є вже схожі файли? → `just search "ключове слово"`
2. Які сцени відбуваються поруч? (для локацій)
3. Які події пов'язані? (для історичних локацій)

### Крок 3: Створити файл

**Для локації (`world/<slug>.md`):**

Frontmatter: `type`, `era`, `first-appearance`

Тіло:
```markdown
## Опис
Зовнішній вигляд, атмосфера, розмір, ключові особливості.

## Географія
Де знаходиться відносно інших локацій. Клімат, ландшафт.

## Історія
Як виникла, що тут відбувалося. Wikilinks на [[події]].

## Мешканці
Хто тут живе, які фракції присутні.

## Зв'язки
- [[інша-локація]] — дорога на північ
- [[персонаж]] — родом звідси
```

**Для концепції (`notes/<slug>.md`):**

Тіло:
```markdown
## Суть
Коротке визначення концепції.

## Правила
Як це працює, обмеження, закони.

## Роль у світі
Як впливає на суспільство, персонажів, сюжет.

## Зв'язки
- [[персонаж]] — використовує магію
- [[подія]] — приклад застосування
```

### Крок 4: Зв'язати з існуючим контентом

1. Додай wikilinks у тілі на: пов'язані локації, персонажів, події, сцени (`first-appearance`)
2. Запропонуй оновити існуючі сцени — додати `setting: "[[нова-локація]]"` у frontmatter
3. Якщо локація вже згадується в сценах — запропонуй додати `[[wikilinks]]` навколо назви

## Правила

1. **Локації → `world/`.** Конкретні місця, які можна відвідати: міста, будівлі, ліси, ріки, гори, підземелля.
2. **Концепції → `notes/` з `type: worldbuilding`.** Абстрактні системи: магія, релігія, культура, політика, економіка, технології.
3. **Пов'язана історична подія → `events/`.** Якщо локація має ключову історичну подію — створи окремий файл у `events/`, не описуй подію всередині локації.
4. **Консистентність.** Якщо світ уже має географію — нова локація не має їй суперечити. Перевіряй сусідні локації.
5. **Не дублюй.** Одна локація — один файл. Якщо місто має райони — вони можуть бути підрозділами в тому ж файлі, або окремими файлами якщо масштабні.
6. **Показуй звіт:** що створено, тип, пов'язані файли, додані wikilinks.

## Приклади

### Приклад 1: Нова локація

> **Користувач:** Створи локацію «Темний ліс» на північ від [[old-mill]]. Моторошне місце, де відбувається [[the-ritual]].

**Дії агента:**
1. Перевіряє, чи є вже `world/dark-forest.md` → ні
2. Створює `world/dark-forest.md`:
   ```yaml
   ---
   title: "Темний ліс"
   created: 2026-07-12
   updated: 2026-07-12
   type: natural
   era: ""
   first-appearance: "[[ch02-sc03]]"
   tags: [location]
   ---
   ```
3. Описує: зовнішній вигляд, атмосферу, що росте/живе
4. Географія: «На північ від [[old-mill]], межує з [[village]] на сході»
5. Історія: «Місце проведення [[the-ritual]] з давніх часів»
6. Звітує: «Створено `world/dark-forest.md`, type: natural. Додано wikilinks на [[old-mill]], [[the-ritual]], [[village]].»

### Приклад 2: Магічна система

> **Користувач:** Опиши магічну систему мого світу. Магія залежить від фаз місяця: повний місяць — максимальна сила, новий місяць — магія зникає.

**Дії агента:**
1. Визначає: це концепція → `notes/magic-system.md`, `type: worldbuilding`
2. Створює файл:
   ```yaml
   ---
   title: "Магічна система: Місячний цикл"
   created: 2026-07-12
   updated: 2026-07-12
   type: worldbuilding
   sources: []
   aliases: [місячна магія, фази місяця]
   tags: [magic, worldbuilding]
   ---
   ```
3. Описує правила:
   - Повний місяць: сила x3, доступні всі заклинання
   - Зростаючий місяць: сила зростає
   - Спадний місяць: сила спадає
   - Новий місяць: магія недоступна
4. Роль у світі: «Маги планують битви на повний місяць. [[vlad]] народився в новий місяць — тому не вірить у свою магічну силу.»
5. Звітує: «Створено `notes/magic-system.md`, type: worldbuilding. Додано wikilinks на [[vlad]], [[the-ritual]].»

### Приклад 3: Історія світу

> **Користувач:** Опиши епоху «Темні віки» — період до основної історії, коли магія була поза законом.

**Дії агента:**
1. Це концепція + події → основне в `notes/dark-ages.md`, ключові події окремо в `events/`
2. Створює `notes/dark-ages.md`, type: worldbuilding
3. Описує епоху: хронологія, ключові закони, наслідки для сучасності
4. Пропонує створити пов'язані події: «Велике спалення магів», «Указ про заборону»
5. Звітує з пропозиціями
