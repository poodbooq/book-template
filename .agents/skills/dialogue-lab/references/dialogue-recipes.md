# Dialogue Lab — рецепти та розширені прийоми

Цей файл містить великі приклади й опціональні прийоми для `dialogue-lab`. Основний процес та обов'язкові правила лежать у [../SKILL.md](../SKILL.md).

## Каталог драматургічних beats

Обирай лише ті beats, які змінюють стан сцени.

| Beat | Дія | Коли корисний |
|---|---|---|
| Привід | хтось приносить предмет, звістку або вимогу | потрібен швидкий старт без пояснювального вступу |
| Тест | персонаж ставить небезпечне запитання або робить малу провокацію | треба перевірити лояльність чи знання |
| Ухилення | відповідь є неповною, надто точною або переводить тему | персонаж ще не готовий віддати владу |
| Контроль простору | хтось зачиняє двері, наближається, забирає предмет, готується піти | локація має вплинути на баланс сил |
| Розворот | третій учасник змінює адресата або сенс сказаного | для полілогу, де двостороння сварка застигла |
| Розкриття ціни | учасник називає, що втратить відмовившись або погодившись | щоб вибір був драматичним, а не формальним |
| Часткова поступка | персонаж віддає малу правду чи ресурс, але лишає головне при собі | якщо повне примирення передчасне |
| Вихід | хтось іде, робить обіцянку, лишає предмет або викликає іншого | щоб створити тиск для наступної сцени |

## Рецепт: заданий фінал (`destination`)

### Вхід

```text
Локація: [[old-mill]], пізній вечір, дощ.
Учасники: [[marta]], [[vlad]], [[ostap]].
Ввідні: Марта дізналась, що Влад приховав від неї лист.
Початковий стан:
- Марта: стримує лють, хоче почути правду.
- Влад: боїться втратити її довіру, але не готовий видати все.
- Остап: удає посередника, але хоче використати конфлікт.
Бажаний стан: Марта не пробачає Влада, але погоджується піти з ним
до [[dark-forest]]; Остап отримує причину діяти самостійно.
Формат: напружена мінісцена, 700 слів.
```

### Перевірка траєкторії

Перед повним текстом переконайся, що кожен перехід має причину:

```text
недовіра → нова спільна загроза → тимчасовий союз
```

Якщо перехід неможливий, не переписуй характер персонажа. Додай правдоподібний місток: доказ, ставку, умову угоди або зовнішній тиск.

### Очікуваний результат

- Марта не «вибачає», а робить тактичний вибір.
- Влад платить частковою правдою, ризиком або поступкою.
- Остап не пояснює свій план вголос; отримує важіль через дію чи недомовленість.

## Рецепт: пошук фіналу (`exploration`)

### Вхід

```text
Дай полілог [[marta]], [[vlad]] і [[ostap]] на кухні після пожежі.
Остап прийшов нібито мирити їх, але сам хоче використати конфлікт.
Не знаю, чим це має закінчитися; хочу знайти наступну сцену.
```

### Робочий вибір

Не роби трьох рівних голосів. Задай функції:

| Учасник | Функція в сцені |
|---|---|
| Марта | тисне на правду |
| Влад | захищає таємницю або союз |
| Остап | змінює тему, адресата і баланс сил |

Побудуй одну повну траєкторію, наприклад: Остап отримує лист або чує фразу, що дає йому привід діяти окремо. Після неї покажи ще 2–3 короткі можливості для наступної сцени.

## Рецепт: короткий діалог для пошуку голосу

### Вхід

```text
Напиши лише 20 реплік сварки між [[vlad]] і [[ostap]] через спадок.
Без пояснень і без збереження у файл.
```

### Правила відповіді

- Не додавай карту напруги, аналіз або метадані.
- Дай лише репліки та мінімальні дії між ними.
- Нехай кожна репліка змінює тиск, а не повторює попередню претензію.
- Не пропонуй записувати текст у `scenes/`, якщо користувач цього не просив.

## Формати виходу

### Повна лабораторна чернетка

```markdown
## Режим
`destination`

## Робочі припущення
- …

## Карта напруги
| Учасник | Ціль | Страх / потреба | Важіль |
|---|---|---|---|

## Чернетка діалогу
…

## Що змінилося
- …

## Сюжетні можливості
1. …
```

### Варіанти без повних сцен

```markdown
## Варіант A — тимчасовий союз
Стан наприкінці: …
Ціна: …
Наступна сцена: …

## Варіант B — розрив
…
```

Залиш повний текст для обраного варіанта або коли користувач окремо попросив його написати.
