# Схема даних

Усі нотатки — Markdown-файли з YAML frontmatter. Slug дорівнює назві файла без `.md` і має бути унікальним у всьому проєкті.

## Спільні поля

| Поле | Тип | Значення |
|---|---|---|
| `title` | string | Людська назва нотатки |
| `created` | `YYYY-MM-DD` | Дата створення файла в реальному світі |
| `updated` | `YYYY-MM-DD` | Дата останнього змістовного оновлення |
| `aliases` | list[string] | Альтернативні назви для пошуку, де передбачено шаблоном |
| `tags` | list[string] | Теги без `#` |

## Статуси рукопису

Сцени й глави мають рівно один статус:

- `draft` — не експортується;
- `revised` — експортується, але ще редагується;
- `final` — експортується і вважається завершеним.

Невідомий або відсутній статус є помилкою. `just check` перевіряє це до збірки.

## Сцена

Обов'язкові поля: `status`, `word-count`, `pov`, `setting`, `story-date`, `timeline-order`, `act`.

- `story-date` — дата або зрозумілий маркер часу **всередині історії**.
- `timeline-order` — ціле число для машинного сортування сцен у хронології.
- `pov` і `setting` можуть містити wikilinks.

## Глава

Обов'язкові поля: `status`, `order`, `act`, `part`, `pov`.

`order` — унікальне ціле число, яке визначає порядок глав. Тіло глави містить лише впорядкований список звичайних zk-сумісних посилань на сцени:

```markdown
- [[opening-scene]]
- [[meeting-at-mill]]
```

`![[...]]` не використовується, бо `zk` не індексує Obsidian embeds як backlinks.

## Подія

- `created`/`updated` — реальні дати роботи над нотаткою;
- `story-date` — дата події у світі книги;
- `order` — числовий порядок, якщо календар світу не дає зручного сортування;
- `era`, `type`, `location`, `characters`, `causes`, `consequences` — сюжетні метадані.

## Research та worldbuilding

Нотатки в `notes/` використовують `type` (`research`, `reference`, `worldbuilding`, `idea`) і `sources` як список URL або бібліографічних посилань.

## Wikilinks та експорт

- `[[slug]]` показує `title` цільової нотатки;
- `[[slug|текст]]` показує явно заданий текст;
- у прозі посилання додається лише на перше входження сутності;
- під час експорту технічна розмітка прибирається;
- broken wikilinks блокують збірку опублікованого рукопису.
